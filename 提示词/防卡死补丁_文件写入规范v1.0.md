# Antigravity 文件写入防卡死补丁 v1.0

> **问题**：Antigravity 内置的 write_to_file 和 replace_file_content 工具在写入超过500个中文字符时会超时卡死，导致章节写入、工作流修改等操作频繁中断。
>
> **适用范围**：所有使用 Antigravity Agent 进行中文长文本创作的项目。
>
> **使用方法**：将此文件发送给其他用户，让他们在 Antigravity 对话框中输入：`请阅读并执行这个补丁文件：[粘贴本文件路径]`

---

## 补丁内容

请 Agent 依次执行以下 3 个步骤：

### 步骤1：创建全局知识项（一次性，所有项目生效）

请在终端执行以下 PowerShell 命令，创建防卡死知识项：

```powershell
 = Join-Path C:\Users\46541 ".gemini\antigravity\knowledge\file_write_anti_freeze"
New-Item -Path  -ItemType Directory -Force | Out-Null
New-Item -Path "\artifacts" -ItemType Directory -Force | Out-Null

 = '{"summary":"文件写入防卡死规范：禁止内置工具写入大块中文文本，统一使用PowerShell分段写入","created":"2026-04-14T00:00:00Z","lastModified":"2026-04-14T00:00:00Z","references":[]}'
[System.IO.File]::WriteAllText("\metadata.json", , [System.Text.Encoding]::UTF8)

 = @'
# 文件写入防卡死全局规范

## 问题根因
Antigravity 内置的 write_to_file 和 replace_file_content 工具，
在处理超过500个中文字符的内容时，极易出现超时/卡死。

## 强制规则

### 新建文件（大文本 > 300字）
分2-3段写入，每段不超过1500字：
- 第1段：[System.IO.File]::WriteAllText("路径", , [System.Text.Encoding]::UTF8)
- 后续段：[System.IO.File]::AppendAllText("路径", , [System.Text.Encoding]::UTF8)

### 修改已有文件（大改动 > 300字）
用 ReadAllText + Replace + WriteAllText：
 = [System.IO.File]::ReadAllText("路径", [System.Text.Encoding]::UTF8)
 = .Replace("旧内容", "新内容")
[System.IO.File]::WriteAllText("路径", , [System.Text.Encoding]::UTF8)

### 安全阈值
- write_to_file：仅用于 < 300字 的小文件
- replace_file_content：仅用于 < 300字 的小改动
- 超过300字：一律走 PowerShell run_command
'@
[System.IO.File]::WriteAllText("\artifacts\file_write_rule.md", , [System.Text.Encoding]::UTF8)
Write-Host "步骤1完成：全局知识项已创建"
```

### 步骤2：为当前项目的续写工作流打补丁（如有）

如果当前项目存在 `.agent/workflows/一键AI续写章节_v4.0.md` 文件，请执行：

```powershell
 = Join-Path (Get-Location) ".agent\workflows\一键AI续写章节_v4.0.md"
if (Test-Path ) {
     = [System.IO.File]::ReadAllLines(, [System.Text.Encoding]::UTF8)
     = False
     = [System.Collections.ArrayList]::new()
    foreach ( in ) {
        [void].Add()
        if (-not  -and  -match "步骤7.*生成与保存") {
            [void].Add("")
            [void].Add("> [!CAUTION]")
            [void].Add("> **文件写入防卡死规范**：禁止使用 write_to_file / replace_file_content 写入超过500字的中文内容！")
            [void].Add("> 必须使用 PowerShell 分段写入：第1段用 WriteAllText 创建，后续用 AppendAllText 追加。")
            [void].Add("> 修改已有文件时，也必须用 PowerShell 的 ReadAllText + Replace + WriteAllText 完成。")
            [void].Add("")
             = True
        }
    }
    [System.IO.File]::WriteAllLines(, .ToArray(), [System.Text.Encoding]::UTF8)
    Write-Host "步骤2完成：工作流已打补丁"
} else {
    Write-Host "步骤2跳过：未找到续写工作流文件"
}
```

### 步骤3：验证补丁是否生效

```powershell

} else {
    Write-Host "[SKIP] 无续写工作流，无需检查"
}
Write-Host ""
Write-Host "=== 补丁安装完成 ==="
```

## 补丁效果

安装后，Agent 在所有项目中写入大文本时将自动遵循以下规则：

| 操作类型 | 禁止方式 | 正确方式 |
|----------|----------|----------|
| 新建大文件（>300字） | write_to_file | PowerShell WriteAllText + AppendAllText |
| 修改大文件（>300字） | replace_file_content | PowerShell ReadAllText + Replace + WriteAllText |
| 小文件（<300字） | - | 内置工具或 PowerShell 均可 |
