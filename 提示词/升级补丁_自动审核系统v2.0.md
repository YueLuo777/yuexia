# 🔧 工业化升级补丁 v2.0 —— 自动审核系统注入

> **适用范围**：所有使用 `.agent/workflows/一键生成细纲.md` 和 `一键AI续写章节_v4.0.md` 的旧项目
> **执行方式**：将本文件放在项目根目录下，拖入 Agent 对话框即可一键打补丁
> **补丁内容**：为细纲生成和正文续写工作流注入自动逻辑审核+修复步骤，并根据项目类型生成专属审核规则

---

## ⚠️ 执行前检查

AI 在执行前必须先确认以下条件：
1. 项目根目录下存在 `.agent/workflows/` 目录
2. 存在 `一键生成细纲.md` 和 `一键AI续写章节_v4.0.md` 两个工作流文件
3. 存在 `00_核心设定/创意白皮书.md` 或类似的白皮书文件（用于识别小说类型）

如缺少以上任一文件，提示用户并中止。

---

## 补丁步骤

### 步骤1: 读取项目信息

读取 `00_核心设定/创意白皮书.md`（或 `00_核心设定/整书大纲.md`），提取以下信息：
- **书名**
- **故事类型标签**（如：官场、重生、年代、修仙、古言、宫斗、都市、甜宠、军事、谍战……）
- **特殊设定**（如独有的功法体系、架空规则等）

输出确认：
```
=== 项目信息确认 ===
书名：[书名]
识别类型：[类型1] + [类型2] + [类型3]
====================
```

### 步骤2: 生成项目专属审核规则文件

根据步骤1识别的类型标签，从下方【审核规则库】中选取对应规则，合并生成 `00_核心设定/审核规则.md`。

**合并公式**：
```
专属审核规则 = 通用规则(T1-T8) + 类型专属规则 + 跨类型补充(按需) + 项目专属自定义项
```

**类型→规则映射**：

| 小说类型组合 | 选取规则编号 |
|-------------|------------|
| 官场/权谋 | GC1-GC5 |
| 历史/年代 | LS1-LS5 |
| 修仙/玄幻 | XX1-XX6 |
| 都市/商战 | DS1-DS5 |
| 军事/谍战 | JS1-JS5 |
| 系统/游戏 | XT1-XT4 |
| 末日/废土 | MD1-MD3 |
| 悬疑/推理 | XY1-XY4 |
| 古代言情/宫斗 | GY1-GY6 |
| 现代言情/甜宠 | XA1-XA6 |
| 穿越/重生 | CZ1-CZ4 |
| 女尊/女强 | NQ1-NQ3 |
| 耽美/百合 | DM1-DM3 |
| 含感情线 | QG1-QG3 |
| 含金手指/外挂 | GUA1-GUA3 |

生成的 `00_核心设定/审核规则.md` 格式：

```markdown
# [书名] 专属审核规则

> 本文件由升级补丁自动生成，供 一键生成细纲(步骤4.5) 和 一键AI续写章节(步骤8.5) 读取使用。

## 通用规则
[从规则库粘贴T1-T8]

## 类型专属规则 — [类型名]
[从规则库粘贴对应规则]

## 跨类型补充规则
[按需粘贴]

## 项目专属检查项
[根据白皮书中的独特设定追加]
```

### 步骤3: 升级「一键生成细纲.md」

打开 `.agent/workflows/一键生成细纲.md`，检查是否已存在"步骤4.5"。

**如果不存在**，在"步骤4: 节奏检查"之后、"步骤5: 保存细纲"之前，插入以下内容：

```markdown
### 步骤4.5: ⚡自动逻辑审核与修复（生成完成后强制执行）⚡

> [!CAUTION]
> **此步骤在步骤4通过后自动执行，不得跳过。**
> 审核不通过的细纲必须自动修复后才能保存。

#### 4.5.1 读取审核规则
读取 `00_核心设定/审核规则.md`，获取本项目的全部审核检查项。

#### 4.5.2 执行审核
对刚生成的5章细纲，按审核规则逐一检查。

#### 4.5.3 自动修复
1. 按检查项逐章扫描，列出所有问题
2. 对每个问题标注严重度（🔴严重/🟡中度/🟢轻度）
3. 对🔴和🟡问题，**直接在细纲文件中修复**，无需等待用户确认
4. 对🟢问题，标注但不做修改（留给正文阶段处理）
5. 修复完成后，输出简要的审核报告

#### 4.5.4 输出审核结果

\```
=== 细纲逻辑审核报告 ===
审核范围：第[X]-[Y]章细纲
发现问题：🔴[X]个 🟡[X]个 🟢[X]个
已自动修复：[X]个
待正文阶段处理：[X]个

[问题1] 🔴 [章节] [问题描述] → 已修复：[修复措施]
[问题2] 🟡 [章节] [问题描述] → 已修复：[修复措施]
[问题3] 🟢 [章节] [问题描述] → 待处理
========================
\```
```

**如果已存在**，跳过此步骤并提示"细纲审核步骤已存在，跳过"。

### 步骤4: 升级「一键AI续写章节_v4.0.md」

打开 `.agent/workflows/一键AI续写章节_v4.0.md`，检查是否已存在"步骤8.5"。

**如果不存在**，在"步骤8: 质量自检"的表格之后、"步骤9"之前，插入以下内容：

```markdown
### 步骤8.5: ⚡自动逻辑审核与修复（正文生成后强制执行）⚡

> [!CAUTION]
> **此步骤在步骤8的质量自检全部通过后自动执行，不得跳过。**
> 审核发现的🔴🟡问题必须自动修复后才能保存最终文件。

#### 8.5.1 读取审核规则
读取 `00_核心设定/审核规则.md`，获取本项目的全部审核检查项。

#### 8.5.2 执行审核
对刚生成的正文，按审核规则中的每一条逐段检查，额外增加以下正文专属检查：
- 破折号残留检查（——/---/—）
- 繁简体混用检查
- 重复表达检查（高频比喻/句式/动作）

#### 8.5.3 自动修复
1. 按检查项逐段扫描，列出问题并标注严重度（🔴/🟡/🟢）
2. 对🔴🟡问题直接修复，🟢问题标注
3. 修复后重新验证字数（确保≥2200字红线）
4. 输出简要审核报告

\```
=== 正文逻辑审核报告 ===
审核章节：第[X]章
发现问题：🔴[X]个 🟡[X]个 🟢[X]个
已自动修复：[X]个
修复后字数：[X]字 ✅/❌
========================
\```
```

**如果已存在**，跳过此步骤并提示"正文审核步骤已存在，跳过"。
