## 🔢 v4.0 核心更新：字数验证机制

### 字数计算标准
**网站字数** = 所有非空白字符（含中文、数字、字母、标点符号）

**排除项**：
- 空格、换行符、制表符
- Markdown 格式符号（如 # * ` 等）

### 字数验证命令
// turbo
```powershell
$content = Get-Content -Path "目标文件路径" -Raw -Encoding UTF8
$noWhitespace = $content -replace '[\s\r\n]', ''
$totalCount = $noWhitespace.Length
Write-Host "总字数(含标点): $totalCount"
```

> ⚠️ **注意**：网站统计方式一般包含标点符号，因此本命令统计所有非空白字符。

| 高潮章节 | 2200字 | 2500-3000字 | 3500字 |

## 使用方法与自动探测机制

1. **指定指令**: 明确指定需要续写的章节（例：`@[/一键AI续写章节_v4.0] 续写第41章`）。
2. **自动探测 (无参触发)**: 若用户仅单独调用工作流指令（如 `@[/一键AI续写章节_v4.0]`）而未做任何指定或说明，AI 必须强制执行以下动作：
   - 自动扫描项目内的 `正文` 文件夹及子卷目录，找到当前最高/最新的正文记录文件（例：第40章）。
   - 根据找出的最新正文章节号，自动向后顺延，将接下来的 **连续5章**（例如41-45章）作为全部任务目标。
   - 自动前往细纲库检索这5章的细纲资料。
   - 然后**顺序逐章**为你连续写完这5章正文（在此过程中，针对每一章仍需严格遵守多段落生成、拼接、扩写、字数强制验证等流程，确认一章合格后再进行下一章的生成和保存）。

### 步骤2: 分段生成策略（v4.0核心）

**问题**：一次性生成容易字数不足。

**解决方案**：分5个部分生成，每部分独立验证字数。

```
┌──────────────────────────────────────────┐
│ 第1部分：开头 (400-500字)                │
│ • 与前章结尾完美衔接                       │
│ • 设定场景、人物状态                       │
│ • 引出本章核心事件                         │
├──────────────────────────────────────────┤
│ 第2部分：前段 (400-600字)               │
│ • 铺垫本章核心冲突                         │
│ • 引入新信息或对话                         │
│ • 1-2个小转折                             │
├──────────────────────────────────────────┤
│ 第3部分：中段 (500-800字)               │
│ • 冲突爆发/主要剧情推进                    │
│ • 主角行动+系统反馈（如适用）               │
│ • 2-3个小转折 + 1个爽点                   │
├──────────────────────────────────────────┤
│ 第4部分：高潮 (400-600字)               │
│ • 爽点释放/高潮                           │
│ • 主角内心反应                            │
│ • 重要决定或转折                           │
├──────────────────────────────────────────┤
│ 第5部分：结尾 (300-500字)                 │
│ • 收尾本章剧情                            │
│ • 为下章埋线                              │
│ • 悬念/期待/转折钩子                       │
└──────────────────────────────────────────┘

总计目标：2200-3000字（标准章节，弹性可至3500字）
```

***

### 步骤3: 生成流程

#### 3.1 分段生成
1. 按上述5个部分依次生成
2. 每部分生成后立即检查字数
3. 如单部分字数不足，立即补充

#### 3.2 合并与验证
1. 将5部分合并为完整章节
2. 执行字数验证命令
3. 如总字数 < 2200，进入强制扩写

#### 3.3 强制扩写循环
如果字数不达标：
1. 识别可扩写的场景（对话、心理、环境）
2. 插入以下内容类型：
   - 增加对话来回（每组+100字）
   - 增加内心吐槽（每段+80字）
   - 增加配角反应（每人+60字）
   - 增加系统提示（每条+50字）
3. 重新验证，直到达标

#### 8.5.1 正文标准审核清单

| # | 检查维度 | 检查内容 | 常见问题 |
|---|----------|----------|----------|
| 1 | **时间线连贯** | 正文时间节点是否与`时间线发展记录.md`一致？人物年龄是否正确？ | 年龄错误、季节矛盾 |
| 2 | **时代物品检查** | 出现的物品、科技、通信工具是否符合章节所处年代？ | 不符年代的物品出现 |
| 3 | **与前章衔接** | 开头是否与前一章结尾剧情、人物状态、时间地点连贯？ | 前章人在A地，本章突然在B地 |
| 4 | **与细纲一致** | 正文是否覆盖了细纲中的所有核心事件点？ | 遗漏核心事件 |
| 5 | **法律/程序合规** | 涉及执法、司法的情节是否有基本的程序包装？ | 赃款直接分发、无证抓人 |
| 6 | **人物称谓一致** | 同一人物在不同段落/章节的称谓是否一致？ | 两章同一角色称呼混淆 |

| 8 | **繁简体检查** | 是否有繁体字混入？ | "這""們"等 |

4. 输出简要审核报告
