# 一键毒点检测与修改工作流

## 使用方法
1. 指定要检测的章节文件路径
2. AI 自动分析并生成修改建议
3. 在 `正文_修改版/` 创建副本并进行修改
4. 章末附【修改记录】说明

## 执行步骤

### 步骤1: 读取章节原文
读取指定的章节 Markdown 文件

### 步骤2: 读取检测规则
**毒点检测规则**：
1. **节奏拖沓**: 超过500字无冲突、对话或信息增量
2. **主角人设崩塌**: 行为与核心人设矛盾（参考全局创作规范）
3. **爽点缺失**: 连续超过1500字无爽点（打脸、升级、逆转等）
4. **逻辑硬伤**: 前后矛盾、不符合已有设定
5. **章末无钩子**: 结尾平淡无悬念
6. **行文规范违反**:
   - 出现括号吐槽或解释（系统提示除外）
   - 引号滥用（对话以外使用引号）
   - 过多修饰词和比喻
   - AI痕迹过重

### 步骤3: 执行检测
逐段分析，标记所有毒点位置及原因

### 步骤4: 生成修改方案
为每个毒点提供具体修改建议

### 步骤5: 创建修改副本
1. 在 `正文_修改版/` 目录下创建同名副本
2. 在副本中实施修改
3. 在章末添加修改记录

## 修改记录格式

```markdown
---

## 【修改记录】

### 修改时间
YYYY-MM-DD HH:MM

### 修改内容

#### 修改1: [位置描述]
- **原文**: "..."
- **修改后**: "..."
- **修改原因**: [具体原因]

#### 修改2: ...

### 修改统计
- 共检测到毒点: X 处
- 已修改: X 处
- 保留原文: X 处（原因: ...）
```

## 常见毒点修改示例

### 节奏拖沓
- 删减冗余描写
- 增加对话推进剧情
- 插入冲突或转折

### 主角人设崩塌
- 调整内心独白体现真实想法
- 用行动反映核心性格
- 增加吐槽和自我解嘲

### 爽点缺失
- 增加误解/迪化场景
- 添加打脸桥段
- 设计阴差阳错的逆袭

### 章末无钩子
- 留下悬念（身份危机、新威胁）
- 设置下章期待（福利暗示、对手出场）
- 情绪高点戛然而止
