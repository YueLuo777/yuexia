# 全局创作规范

## 适用范围
本规范适用于《{{书名}}》所有章节的创作，包括AI续写和人工写作。

---

## 一、字数规范

- **每章字数**: 2200-3000字
- **弹性范围**: 高潮章节可达3500字
- **硬性下限**: 每章不得低于2200字（硬性检测红线）

---

## 二、行文规范

### 2.1 禁止使用
1. **括号吐槽/解释**: 不要使用（这样的括号注释）来进行吐槽或解释
   - ✗ 错误: 主角心想（这不是坑爹吗）
   - ✓ 正确: 主角心里骂娘，这不是坑爹吗？

2. **引号滥用**: 引号仅用于对话内容
   - ✗ 错误: 这是所谓的"智谋"
   - ✓ 正确: 这是所谓的智谋

3. **系统提示例外**: 只有"正经"的系统提示可以使用【】或特殊格式
   - ✓ 允许: 【系统提示：XXX】

### 2.2 文风要求
1. **以对话为主**: 对话应占正文70%以上
2. **减少描写**: 避免大段环境/心理描写，用对白推进
3. **去AI痕迹**: 
   - 避免"首先...其次...最后"式总结
   - 避免过多解释性语句
   - 避免形容词堆砌
4. **贴近原文**: 参考已有章节风格

### 2.3 对话风格
- 主角对话要有个性
- 其他角色说话要符合身份
- 避免每个人都一个调

---

## 三、结构规范

### 3.1 章节结构
```
开头(300字): 承接上章，快速进入场景
前1/3: 铺垫冲突/建立期待
中1/3: 冲突爆发，主角被迫行动
后1/3: 结果揭晓，爽点释放
结尾(200字): 留钩子
```

### 3.2 章末钩子（必须）
每章结尾必须有钩子，类型包括:
- 悬念型: 身份危机、新威胁
- 期待型: 即将获得好处
- 转折型: 情况突变
- 信息型: 透露关键信息
- 冲突型: 新对手出场

### 3.3 爽点节奏
- 每500字应有一个小转折或信息点
- 每1500字应有一个爽点释放
- 每章至少一个高光时刻

## 六、番茄爆款要素

### 6.1 必备要素
- **强代入感**: 读者能代入主角处境
- **即时反馈**: 主角行动快速得到结果
- **打脸爽感**: 看不起主角的人被打脸
- **持续期待**: 不断有新目标和新危机

### 6.2 节奏参考
- **开场5章**: 建立世界观+核心矛盾
- **每10章**: 一个小高潮
- **每卷末**: 一个大高潮
- **全书**: 持续升级+最终反转

### 6.3 禁止雷区
- 连续超过2章无爽点
- 主角长时间无作为
- 剧情陷入僵局
- 配角戏份过多（每章主角出场必须超过70%）

---
