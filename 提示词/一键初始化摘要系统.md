# 一键初始化摘要系统

> **用途**：在新的小说项目中快速部署"前文摘要管理系统"，解决长篇小说创作时"写着写着就忘了前文"的问题。

---

## 执行步骤

### 步骤1: 创建目录结构
// turbo
创建以下目录（如已存在则跳过）：
```
09_剧情摘要库/
├── 卷一_[卷名]/     # 根据项目实际卷名创建
└── (后续卷按需创建)
```

### 步骤2: 创建摘要生成模板
// turbo
在 `09_剧情摘要库/摘要生成模板.md` 创建标准模板

### 步骤3: 创建全书剧情主线初始文件
// turbo
在 `09_剧情摘要库/00_全书剧情主线.md` 创建空白模板

## 摘要类型说明

### 章节摘要（300-500字/章）
- 记录每章关键剧情点
- 最细粒度的摘要

### 分段摘要（每10章汇总，800-1200字）
- 快速回顾阶段发展
- 续写时主要参考

### 卷摘要（1000-2200字/卷）
- 概括整卷内容
- 细纲生成时主要参考

### 全书剧情主线
- 记录整部小说核心脉络
- 最宏观的参考

---

## 摘要生成原则

### ✅ 必须包含
1. **核心剧情点**：本章最重要的1-3个事件
2. **人物变化**：身份、关系、心态的任何变化
3. **伏笔信息**：明确标注埋设和回收的伏笔
4. **悬念/钩子**：章末留下的悬念

### ❌ 避免
1. 流水账式的全文复述
2. 无关紧要的场景描写
3. 重复已在人物档案中记录的信息
4. 过于主观的评价

新章节完成 → /一键状态更新 → 自动触发摘要生成 → 摘要库更新
