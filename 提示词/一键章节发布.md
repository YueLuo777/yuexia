# 一键章节发布工作流

## 使用方法
1. 完成章节创作后调用此工作流
2. AI 自动执行所有收尾工作
3. 输出发布报告

## 执行步骤

### 步骤0: 自动寻章与锁定（缺省模式）
- **触发条件**：当用户仅输入 `/一键章节发布` 或在调用此工作流时未指定任何具体章节号，必须触发此逻辑。
- **自动定位逻辑**：
  1. 扫描本项目储存正文内容的文件夹。
  2. **跨卷检索**：若正文以不同卷宗文件夹划分，则按照卷的序号或顺序，自后向前排查，精准定位到**最新且内部确实有正文文件的那个卷**。
  3. **定位最新五章**：在该最新卷文件夹内，按章节序号进行倒序排列，自动提取**最新的五章正文**（若该卷当前所有章节数不足五章，则提取全部）。
  4. 将截获的这五章（或符合条件的全部最新章节）锁定为本次发布流程的执行目标文件。
- **指令优先**：如果用户在调用时明确指定了目标章节（例如： `/一键章节发布 最新一章` 或 `发布第X至Y章`），则忽略此自动定位步骤，严格按照用户的指定执行。

### 步骤1: 格式规范检查
检查章节文件是否符合以下规范:

**结构检查**:
- [ ] 标题格式: `第X章 章节名`（无#号）
- [ ] 正文内容完整
- [ ] 章末不能有类似 `（本章完）`这样与正文无关的内容

**行文检查**:
- [ ] 无括号吐槽（系统提示除外）
- [ ] 无引号滥用
- [ ] 无过多修饰词
- [ ] 对话占主体
- [ ] 章末有钩子

**字数检查**:
- [ ] 字数在2200-3000字范围内（弹性可至3500字，硬性不低于2200字）

### 步骤2: 文件命名规范
确保文件名格式正确:
- 格式: `00XXX第X章 章节名_XXXX字.md` 或 `00XXX第X章 章节名.md`
- 5位数字序号 + "第X章" + 空格 + 章节名 + (.md)

如命名不规范，自动重命名

### 步骤3: 状态与设定库更新
深入分析最新章节内容，提取并更新以下设定库信息:

**1. 人物相关 (`01_人物列表库/`)**:
- 提取: 新登场人物、位置变化、职务/身份变化、人际关系变化、心态转变
- 更新格式:
```markdown
## 状态更新记录
### [第X章] - YYYY-MM-DD
- **变化内容**: [具体描述]
- **章节位置**: 第X章 "对话/场景描述"
```

**2. 势力相关 (`02_势力设定库/`)**:
- 提取并更新: 势力实力消长、势力间关系变化、核心成员变动

**3. 地图/物品相关 (`03_地图库/`, `05_重要物品库/`)**:
- 提取并更新: 新地点出现/控制权更迭、新物品获得/消耗

**4. 伏笔状态 (`07_伏笔库/`)**:
- 新埋伏笔添加到 `已埋伏笔.md`，已回收伏笔移动到 `已收伏笔.md`，并更新总表

**5. 时间线发展记录 (`00_核心设定/时间线发展记录.md`)**:
- **提取规则**：无论时间是否有大跨度跳跃（如跨年/换季），在执行单章或批量发布工作流时，**必须强制**提炼出该阶段的核心进展，作为一个独立的连载时间线节点。
- **更新动作**：无需判断“是否发生时间变更”，请直接在 `时间线发展记录.md` 最下方的“剧情推进记录”区域追加新的节点，格式如下：
  ```markdown
  ### [第X-Y章更新] 年份+季节/月份（如：1986年冬初）
  - **阶段性事件**: [简明扼要提炼本批次章节的核心大事件、势力大洗牌、核心冲突结果。]
  ```

最后，在 `08_更新记录/` 中追加本次全局设定的更新摘要日志。

### 步骤4: 生成与更新剧情摘要
基于 `09_剧情摘要库/摘要生成模板.md` 规范，对本章进行摘要提取：
- **必须提取**: 1-3个核心事件、人物动态、关键线索、经典台词、章末结局和悬念。
- **更新分段摘要**: 汇总本章关键信息，写入 `09_剧情摘要库/卷X_卷名/第XXX-XXX章摘要.md`。
- **更新大卷与总干线**: 
   - 检查章节是否为新卷起始或结束，更新 `卷摘要.md`。
   - 若发生核心重大变化(主角身份大跃迁、核心反派退场等)，同步精简更新 `09_剧情摘要库/00_全书剧情主线.md`。

### 步骤5: 生成发布报告
综合输出本章的终极发布报告:

```markdown
# 第X章 发布报告

## 基本信息
- **章节名**: 
- **真实字数**: X字
- **所属卷**: 

## 规范检查结果
- 格式检查: ✓ 通过 / ✗ [问题描述]
- 行文检查: ✓ 通过 / ✗ [问题描述]  
- 字数检查: ✓ 通过 / ✗ [问题描述]

## 状态更新摘要
### 人物、势力与时间线更新
- **时间线推进**: [有/无] [具体描述]
- **[人物/势力名]**: [变化描述]

### 剧情摘要提炼
- **核心事件**: [一两句话概括]
- **章末悬念**: [简述]

## 下一章预告
- 预计方向: [简述]
```

## 批量发布模式
支持一次性指定多个章节（如: 第250章-第255章），流程将对指定的每一章依次循环执行规范检查、设定库更新、摘要提取等全部步骤，最终输出汇总版报告。
