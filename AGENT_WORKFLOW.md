# AI 代码修改工作流规范

> 每次修改代码前必须阅读此文件，修改完成后必须逐项自检，确认无误后方可向用户反馈。

---

## 一、修改前必读（Pre-Modification）

### 1. 文件阅读
- [ ] **必须**先读取所有待修改的源文件，完整理解当前代码逻辑
- [ ] 如涉及多个文件联动，**全部读取**后再动手，禁止边读边改
- [ ] 通过搜索（`grep`）确认目标字符串/变量名在文件中的**所有出现位置**
- [ ] 如涉及 UI 字符串（如标题、标签、提示文字），**全文搜索**确认该文字在项目中是否有多处使用

### 2. 需求澄清
- [ ] 确认修改范围：是局部组件还是全局影响？
- [ ] 确认边界情况：空状态、默认值、错误处理
- [ ] 如涉及弹窗/路由/状态，确认与其他页面的联动关系

---

## 二、修改执行（During Modification）

### 1. 精确替换原则
- [ ] 使用 `edit_file` 时必须保证 `old_string` 在目标文件中**唯一且完全匹配**
- [ ] 如涉及字符串重命名，确认不会误伤其他不相关代码
- [ ] 删除代码前先确认该变量/函数在其他地方**无引用**

### 2. 一致性检查
- [ ] 修改 Header 导航文字 → 同步检查 WorkbenchLayout.tsx 弹窗标题
- [ ] 修改页面标题 → 同步检查弹窗 `CenterModal` 的 `title` 属性
- [ ] 修改类型定义 → 同步检查所有使用该类型的组件
- [ ] 修改颜色/样式常量 → 全局搜索确认无遗漏

### 3. 导入清理
- [ ] 删除未使用的 `import`（图标、组件、类型等）
- [ ] 新增 `import` 后确认包已安装且路径正确

---

## 三、修改后强制自检（Post-Modification Checklist）

### 1. 语法与类型检查
- [ ] 执行 `npm run build`（或 `tsc -b`），**必须 0 错误**
- [ ] 如出现 `TS error`，立即修复，禁止跳过
- [ ] 如出现 `Unused import/variable` 警告，立即清理

### 2. 逻辑验证
- [ ] 重新读取修改后的文件，确认替换结果**完全符合预期**
- [ ] 确认没有引入**重复代码**（如相同逻辑复制粘贴后忘记修改）
- [ ] 确认条件分支（`if/else`、三元运算符）逻辑正确
- [ ] 确认状态更新（`setState`、`useEffect` 依赖数组）无死循环或遗漏

### 3. UI/UX 验证（如适用）
- [ ] 确认文字无错别字、无截断、无溢出
- [ ] 确认按钮/输入框交互正常（禁用态、hover 态）
- [ ] 确认弹窗 `z-index` 层级正确，无被遮挡
- [ ] 确认响应式布局在不同宽度下正常

### 4. 数据与持久化验证
- [ ] 涉及 `localStorage` 读写时，确认 key 名称正确、数据格式兼容
- [ ] 涉及向后兼容时（如 storage key 版本升级），确认旧数据迁移逻辑
- [ ] 确认空数据/首次加载的 fallback 行为正确

### 5. 文件级最终审查
- [ ] 执行 `git diff`（或逐段阅读修改后的文件），确认**没有意外修改**
- [ ] 确认文件末尾无多余空行或缺失换行
- [ ] 确认大括号、括号、引号全部正确闭合

---

## 四、向用户反馈规范

**完成全部自检并确认 100% 通过后方可反馈。**

### 反馈内容必须包含：
1. **修改文件清单**：列出所有修改的文件及对应行号
2. **具体变更说明**：每处修改的 "改前 → 改后"
3. **构建状态**：明确说明 `npm run build` 是否通过
4. **已知限制**（如有）：如需要用户刷新缓存、需要后续补充等

### 禁止行为：
- ❌ 未执行 `npm run build` 就告知用户"已完成"
- ❌ 出现编译错误时跳过或隐藏
- ❌ 未核对实际修改结果就凭记忆描述
- ❌ 在存在未修复问题时直接部署

---

## 五、常见问题速查

| 场景 | 检查项 |
|------|--------|
| 修改标题/导航 | 搜索全项目确认所有引用处同步 |
| 修改 CSS/Tailwind | 检查 responsive 前缀（`md:`、`sm:`）是否保留 |
| 修改弹窗内容 | 确认 `z-index`、`onClick` 阻止冒泡、`useEscToClose` 行为 |
| 修改 localStorage | 确认 JSON.parse 有 try-catch，确认 key 无冲突 |
| 删除组件/函数 | 全局搜索确认零引用后再删除 |
| 修改图标按钮 | 确认 `lucide-react` import 已更新，无用图标已移除 |
