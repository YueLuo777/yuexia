# 迁移规则

## 强约束

1. 不修改 `E:\0yuexia\yuexia\` 原项目业务文件
2. 只在 `E:\0yuexia\yuexia\xinyuexia\` 内工作
3. 复用旧代码时，只复制最小必要范围
4. 复制后立即在 `xinyuexia` 内做结构整理，不保留无边界的大文件

## 复制优先级

### 第一批

- 样式文件
- 布局文件
- UI 组件

### 第二批

- 页面壳
- 路由入口
- 基础 hooks

### 第三批

- 复杂业务逻辑
- 拖拽
- 提炼剧情

## 每次迁移的固定流程

1. 记录要复制的来源文件
2. 复制到 `xinyuexia`
3. 拆分并重命名
4. 修正导入路径
5. 验证可解析
6. 验证页面能打开
